# 기획 — 지식 그래프 시각화 (노드 맵 뷰)

> 작성일: 2026-04-13
> 상태: 기획 단계 (미착수)
> 선행 조건: P0.5 지식 그래프 레이어 완료 (`knowledge-graph-prd.md`)
> CAOF 트랙: Critical (새 패키지 도입 + 새 화면)

---

## 1. 목표

현재 텍스트 카드 기반 드릴다운 방식(P0.5)을 보완해, **노드-엣지 네트워크 시각화** 화면을 제공한다. 사용자와 IR 심사위원이 "종 ↔ 환경 ↔ 질병 ↔ 먹이 ↔ 기재" 관계를 한눈에 조망할 수 있게 한다.

### 기대 효과
- IR 시연에서 "AI 기반 지식 그래프" 워딩의 시각적 증거
- 사용자가 종 중심으로 관련 정보를 탐색하는 새로운 UX
- 텍스트 카드 방식과 병행 — 지도에서 노드 탭 → GraphDetailScreen 진입

---

## 2. 화면 설계 (초안)

### 2-1. GraphMapScreen

- **진입점**: 사육 위키 탭 내 "지식 지도" 버튼, 또는 WikiDetailScreen 상단 아이콘
- **초기 뷰**: 선택된 종(species)을 중심 노드로, depth=1~2 범위의 관련 엔티티를 방사형 배치
- **노드**: EntityKind별 색상/아이콘 구분
  - species: 초록 / pets 아이콘
  - env_cond: 파랑 / thermostat
  - disease: 빨강 / medical
  - food: 주황 / restaurant
  - equipment: 회색 / inventory
- **엣지**: RelationType별 라벨 + 색상
- **인터랙션**:
  - 핀치 줌 / 패닝
  - 노드 탭 → 해당 엔티티의 GraphDetailScreen으로 이동
  - 노드 롱프레스 → 해당 노드를 중심으로 그래프 재배치 (포커스 전환)
- **필터**: EntityKind 토글 (예: 질병만 보기, 먹이만 보기)

### 2-2. 유저 체험 시뮬레이션

```
[화면] 사육 위키 → 레오파드 게코 → 상단 "지식 지도" 아이콘 탭
[반응] 화면 전환 → 중앙에 "레오파드 게코" 큰 노드, 주변에 온도/습도/질병/먹이 노드 방사형 배치. 엣지에 "적정 온도", "취약 질병" 등 라벨.
[조작] 핀치 아웃 → 줌인, 패닝으로 이동
[조작] "MBD" 노드 탭
[반응] GraphDetailScreen 진입 (기존 P0.5 화면 재사용)
[감정] "이 종이 뭐에 취약하고 뭘 먹어야 하는지 한눈에 보인다"
```

## 3. 기술 선택지

### 옵션 A: `graphview` 패키지
- pub.dev: `graphview` (force-directed / tree / layered 레이아웃)
- 장점: 구현 빠름, 레이아웃 알고리즘 내장
- 단점: 커스터마이징 제한, 유지보수 활발도 확인 필요

### 옵션 B: `CustomPainter` + 자체 레이아웃
- Force-directed 알고리즘 직접 구현 (또는 경량 포팅)
- 장점: 완전한 디자인 자유도, 패키지 의존성 없음
- 단점: 구현 비용 높음, 성능 최적화 직접 해야 함

### 옵션 C: `flame` 엔진 Canvas
- 게임 엔진 기반이라 인터랙티브 2D에 강함
- 장점: 줌/패닝/애니메이션 기본 지원
- 단점: 앱에 게임 엔진 의존성 추가는 과잉일 수 있음

### 현재 권장: 옵션 A → 프로토타입 후 판단
- `graphview`로 MVP 빌드 → IR 시연에 충분한지 확인
- 부족하면 옵션 B로 전환

---

## 4. 데이터 소스

기존 `GraphRepository`의 API를 그대로 사용:
- `entity(id)` → 노드 데이터
- `outgoing(entityId)` / `incoming(entityId)` → 엣지 데이터
- `neighbors(entityId, depth: 2)` → 표시할 노드 범위

추가 구현 불필요. 렌더링 레이어만 신규.
