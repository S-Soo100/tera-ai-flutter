# 사육장 탭 디자인 개편 Spec

> 작성일: 2026-06-12 · 대상: `/smart-cage` 탭 (`SmartCageScreen`)
> CAOF 트랙: **Standard** (기존 feature UI 재구성, 되돌리기 비용 중간)
> 상태: **승인 대기** — 구현 전 검토용

---

## 0. 대상 화면 현재 구조

`SmartCageScreen` → `_DeviceBody`는 **별도 카드 2개**를 세로로 쌓는다.

```
ListView (padding: 20,0,20,24)
├─ [device 2개+] _DeviceSelector (칩)
├─ ModuleStatusCard      ← 카드 A: 환경/센서 현황 (온습도)
├─ SizedBox(12)
└─ ActuatorControls      ← 카드 B: 액추에이터 제어
```

- **카드 A (`module_status_card.dart`)**: 디바이스명 + 연결배지 / `목표: 24°C / 70%`(흐릿) / `환경 모니터링`(흐릿) / **현재온도·현재습도 큰 박스 2개** / 보조센서 T2 한 줄
- **카드 B (`actuator_controls.dart`)**: `액추에이터` 타이틀 / 팬·히터 / LED전원·밝기± / 워터펌프
- 두 카드 모두 `Container` + `BoxShadow(black 4%)` + `radius 20`, **테두리(border) 없음**

> 참고: `environment_card.dart`(더미 24.5°)는 **어디서도 import 안 되는 레거시** → 이번 작업과 무관, 건드리지 않음.

## 1. 현황1 — 두 카드를 하나로 통합 + 테두리 강조

| | |
|---|---|
| **현황** | 센서 현황 카드와 액추에이터 카드가 별도 박스로 떨어져 있음 |
| **조치** | 하나의 카드 안에 둘 다 넣고, 카드 테두리를 더 잘 보이게 |

**구현 방향**
- `_DeviceBody`에서 두 위젯을 **단일 외곽 카드**로 감싸고, 내부는 구분선(`Divider`)으로 위(현황)·아래(제어) 분리.
- 외곽 카드 데코레이션: 기존 `BoxShadow`에 더해 **`Border.all(color: outlineVariant, width: 1)`** 추가 (테두리 가시성↑). radius 20 유지.
- 내부 두 섹션(`ModuleStatusCard`, `ActuatorControls`)은 **각자 갖고 있던 그림자/배경 Container를 제거**하고 "내용물"만 남긴다 (이중 카드 방지). 두 위젯에 `embedded: true` 같은 플래그를 주거나, 카드 데코를 상위로 끌어올림.

**영향 파일**: `smart_cage_screen.dart`(통합 셸 추가), `module_status_card.dart`·`actuator_controls.dart`(외곽 데코 제거 분기)

## 2. 현황2 — 목표치를 현재치와 한 박스에

| | |
|---|---|
| **현황** | 현재온도·현재습도가 큰 영역 차지, 목표온도·목표습도는 위에 흐릿한 글자로 분리됨 |
| **조치** | 목표치가 현재치와 같은 박스에 들어가 "현재 N / 목표 M"을 한눈에 |

**구현 방향**
- 흐릿한 `목표: 24°C / 70%` 한 줄(`smart_cage_main_target`)을 **제거**하고, 온도/습도 **각 센서 박스 내부**로 목표값을 이동.
- 센서 박스 레이아웃: `[아이콘 라벨] / 현재값(큰 글씨) / 목표값(작은 보조 글씨)`
  ```
  ┌─────────────┐   ┌─────────────┐
  │ 🌡 현재 온도 │   │ 💧 현재 습도 │
  │   24.5°      │   │   68%        │
  │ 목표 24°     │   │ 목표 70%     │
  └─────────────┘   └─────────────┘
  ```

**⚠️ 데이터 제약 (CAOF 데이터 정확성)**
- 목표 온습도는 **telemetry·device 모델에 필드가 없다**. 현재도 `"목표: 24°C / 70%"` **하드코딩 문자열**일 뿐 실제 setpoint가 아니다.
- 이번 작업은 **표시 위치 이동(상수 분리)**까지만. 온도목표·습도목표를 각각 상수로 분리:
  - 신규 키 `smart_cage_target_temp` = `"목표 24°"`, `smart_cage_target_humidity` = `"목표 70%"` (기존 `smart_cage_main_target` 폐기)
- **실제 디바이스 setpoint 연동은 별도 후속** (terra-server 계약 + 모델 필드 추가 필요). spec 범위 밖임을 명시.

**영향 파일**: `module_status_card.dart`(`_SensorBox`에 목표 라인 추가, 헤더 목표줄 제거), `ko.json`

## 5. 현황5 — 액추에이터 버튼을 iOS 제어 패널 스타일로

| | |
|---|---|
| **현황** | 액추에이터 버튼들이 안 예쁨 (가로로 늘어선 알약형 칩) |
| **조치** | 아이폰 제어센터(Control Center) 스타일로 재조정 |

**현재 컨트롤 5종**: 팬(토글) · 히터(토글+잠금) · LED전원(펄스) · LED밝기(± 스테퍼) · 워터펌프(토글)

**iOS 제어센터 스타일 정의 (이 프로젝트 채택안)**
- **2열 그리드 타일** (`GridView`/`Wrap`), 타일은 둥근 사각형(radius 16~20), 넉넉한 패딩.
- **타일 내부**: 좌상단 원형 아이콘 뱃지 + 하단 라벨 + 상태 텍스트(ON/OFF/잠김).
- **ON 상태**: 강조색 채움 (팬·워터펌프=Green `#2E7D32`, 히터=Amber `#FF8F00`), 흰 아이콘/글씨.
- **OFF 상태**: `surfaceContainerHighest` 반투명 배경 + 원색 아이콘 뱃지(연한 톤), 어두운 글씨 — iOS의 "꺼진 토글"과 동일 무드.
- **눌림 피드백**: `AnimatedContainer` 150ms 색 전환 + 살짝 scale(0.97) (기존 busy dot 유지).
- **히터 잠금**: 타일 우상단 자물쇠 아이콘 오버레이 (현재 로직 유지).
- **LED 밝기(±)**: 토글이 아니므로 **1.5~2칸 폭의 넓은 타일**에 `–  [밝기]  +` 스테퍼를 담아 제어센터의 밝기 슬라이더 느낌. (슬라이더 실연동은 범위 밖, ± 버튼 유지)

**제약**
- **기존 명령 발행 로직(`_sendCommand`/`commandUpdatesProvider`/히터 확인·잠금 다이얼로그/LED 디바운스)은 100% 보존**. 이번 작업은 **프레젠테이션 레이어(타일 위젯)만 교체**.
- `CircularProgressIndicator` 금지 규칙 유지 → busy 표시는 기존 `_BusyDot` 재사용.
- 하드코딩 색상 최소화: 강조 Green/Amber는 이미 코드 전반에서 상수로 쓰임(`#2E7D32`/`#FF8F00`) → 동일 상수 사용.

**영향 파일**: `actuator_controls.dart`(칩 위젯 `_ActuatorChip`/`_HeaterChip`/`_LedOnChip`/`_LedBrightnessRow` → 타일 위젯으로 재작성), 레이아웃 행→그리드

## 6. 체험 설계서 (Before → After)

**Before**: 사용자가 사육장 탭에 들어오면 위에 온습도 박스가 큰 자리를 먹는 카드, 그 아래 뚝 떨어진 곳에 회색 알약 버튼들이 줄지어 있다. "목표가 몇이더라?" 하면 카드 맨 위 흐릿한 작은 글씨를 따로 찾아 읽어야 한다. 버튼은 밋밋하고 어디가 켜진 건지 한눈에 안 들어온다.

**After**:
1. **[화면]** 테두리가 또렷한 **하나의 큰 카드**가 보인다. 위쪽은 현황, 아래쪽은 제어 — 구분선으로 나뉘어 "이건 한 사육장의 상태판"이라는 인상.
2. **[화면]** 온도 박스 안에 `24.5°`(큼)와 바로 아래 `목표 24°`(작음)가 같이 있어, **현재가 목표에 가까운지 한눈에** 안다. 습도도 동일.
3. **[조작]** 아래로 눈을 내리면 제어센터처럼 **2열 타일**. 팬을 탭 → 타일이 **회색→초록으로 채워지며** `ON`. 다시 탭 → 초록→회색 `OFF`. (서버 ack까지 busy dot)
4. **[조작]** 히터 타일 탭 → 위험 확인 다이얼로그(기존). 잠겨 있으면 자물쇠가 보이고 탭 시 해제 다이얼로그.
5. **[감정]** "아이폰 제어센터랑 똑같네" — 학습 비용 0. 켜짐/꺼짐이 색으로 명확, 현황과 제어가 한 카드라 시선 이동이 짧다.

## 8. 검증 계획

1. `flutter analyze` 에러 0.
2. device 0개/1개/2개+ 상태별 렌더 확인 (통합 카드가 빈 상태·로딩 스켈레톤과 충돌 없는지).
3. telemetry 정상/센서오류/오프라인 3상태에서 통합 카드 깨짐 없는지.
4. 액추에이터 ON/OFF 토글, 히터 확인·잠금 다이얼로그, LED 디바운스 **기존대로 동작**.
5. 다크모드에서 테두리·타일 색 대비 확인.
