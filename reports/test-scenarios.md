## A. 크리티컬 — 반드시 검증 (검수에서 다수 지적)

### A-1. 위키 종 선택 전달 (5명 지적)
```
시나리오: 홈 → 사육 가이드 카드 → 위키 종 선택 확인
  GIVEN  홈 화면이 표시되어 있다
  WHEN   "펫테일 게코" 사육 가이드 카드를 탭한다
  THEN   사육 위키 탭으로 이동하고, 종 선택 칩이 "펫테일 게코"로 선택되어 있어야 한다
  현재 예상: ❌ FAIL — /wiki만 호출, 항상 레오파드 게코(기본값)
```

```
시나리오: 내 개체 상세 → 사육 위키 이동
  GIVEN  "모찌" (크레스티드 게코) 개체 상세 화면이다
  WHEN   "이 종의 사육 위키 보기" 버튼을 탭한다
  THEN   사육 위키에서 "크레스티드 게코"가 선택된 상태로 표시되어야 한다
  현재 예상: ❌ FAIL — /wiki만 호출, 레오파드 게코 표시
```

```
시나리오: 검색 → 위키 이동
  GIVEN  검색 화면에서 "펫테일"을 검색했다
  WHEN   "펫테일 게코" 결과의 "상세 정보" 버튼을 탭한다
  THEN   사육 위키에서 "펫테일 게코"가 선택된 상태여야 한다
  현재 예상: ❌ FAIL — /wiki만 호출
```

## C. 일반 — 기능 동작 확인

### C-1. 사육 위키 기본 흐름
```
시나리오: 종 선택 + 카테고리 탐색
  GIVEN  사육 위키 탭
  WHEN   "크레스티드 게코" 칩 선택 → "사육장" 카테고리 탭
  THEN   크레스티드 게코의 사육장 정보 표시 (45×45×60cm 세로형, 코르크 튜브 등)
```

```
시나리오: 3종 비교 화면
  GIVEN  사육 위키 탭
  WHEN   "종 비교" 카테고리 탭
  THEN   3종의 온도/습도/크기/먹이 등 비교 테이블 표시
```

### C-2. 내 개체 CRUD
```
시나리오: 개체 등록
  GIVEN  내 개체 탭 (빈 상태)
  WHEN   "개체 등록하기" → 이름 "모찌", 종 "레오파드 게코", 성별 "수컷" → 저장
  THEN   목록에 "모찌 · 레오파드 게코 · ♂" 카드가 표시된다
```

```
시나리오: 체중 기록 추가
  GIVEN  "모찌" 개체 상세 화면
  WHEN   체중 기록 "추가" → 45.5g 입력 → 저장
  THEN   체중 기록 리스트에 "45.5g" 항목이 추가된다
```

```
시나리오: 개체 수정
  GIVEN  "모찌" 개체 상세 화면
  WHEN   수정 아이콘 → 이름을 "모찌2"로 변경 → 저장
  THEN   상세 화면에 "모찌2"가 표시된다
```

```
시나리오: 개체 삭제
  GIVEN  "모찌" 개체 상세 화면
  WHEN   삭제 아이콘 → 확인 다이얼로그 → 삭제
  THEN   내 개체 목록으로 돌아가고 "모찌"가 사라진다
```

### C-3. 자진신고 가이드
```
시나리오: WIMS 절차 확인
  GIVEN  자진신고 탭
  WHEN   10단계 절차를 순서대로 펼친다
  THEN   1. WIMS 접속 → 2. 회원가입 → ... → 10. 확인증 발급
```

```
시나리오: FAQ 확인
  GIVEN  자진신고 탭, FAQ 섹션
  WHEN   "자진신고와 보관신고는 다른 건가요?" 탭
  THEN   답변이 펼쳐진다
```

### C-4. 검색
```
시나리오: 한글명 검색
  GIVEN  검색 화면
  WHEN   "볼파이썬" 입력
  THEN   "볼파이썬" 결과 표시 + "합법" 뱃지 (featured 아님)
```

```
시나리오: 학명 검색
  GIVEN  검색 화면
  WHEN   "Eublepharis" 입력
  THEN   "레오파드 게코" 결과 표시 + "상세 정보" 뱃지 (featured)
```
